### **Amplifica Brand UI/UX Guidelines**  
**1. Brand Identity & Personality**  
- **Tone:** The interface should feel responsible, reliable, and innovative.  
- **Vibe:** "Empatícity", "Cruise speed" (fast and dynamic), and "Honesty partners" (transparent and clear).  
**2. Color Palette** Do not use any colors outside of this exact palette, except for the specific layout background override (#f8f9fc) mentioned in Section 4.  
- **Primary Blue (Light Blue):** #4548FF.  
- **Dark Blue:** #121755.  
- **Primary Yellow (Yellow):** #F7DC4B.  
- **Dark Gray:** #1D1D1F.  
- **Light Gray:** #E1E0E0.  
**⚠️ 3. CRITICAL BLUE USAGE RULES**  
- **No Blue Backgrounds:** Blue (neither Primary Blue nor Dark Blue) can  **NEVER** be used as a general background color for the application, pages, or large container sections.  
- **UI Elements Only:** Blue is strictly reserved for actionable or structural interface elements, such as headers, buttons, active states, borders, and icons.  
**4. Themed Environments & Background Overrides**  
Apply the following strict logic for backgrounds and UI elements:  
- **Light Mode Background:** MUST use the soft off-white #f8f9fc.  
- **Light Mode Text:** Use Dark Gray (#1D1D1F).  
- **Light Mode UI Elements:** MUST use  **Dark Blue** (#121755) for buttons, headers, and icons. Do not use Primary Blue in Light Mode.  
- **Dark Mode Background:** MUST use  **Dark Gray** (#1D1D1F).  
- **Dark Mode Text:** Use White or Light Gray (#E1E0E0).  
- **Dark Mode UI Elements:** MUST use  **Primary Blue** (#4548FF) for buttons, headers, and icons.  
- **Accents (Both Modes):** Use Primary Yellow (#F7DC4B) sparingly for secondary call-to-actions, warnings, or small highlights.  
**5. Logo Assets, Placement & Handling**  
- **Placement:** The logo must be rendered on the  **top left side** of the app interface.  
- **Light Mode Logo:** `Assets/amplifica-light-mode.png` (local file in amplifica-portal/Assets/)
- **Light Mode Rendering:** Render normally; use on light/white backgrounds.
- **Dark Mode Logo:** `Assets/amplifica-dark-mode.png` (local file in amplifica-portal/Assets/)
- **Dark Mode Rendering:** Render normally; use on dark (Dark Blue #121755) backgrounds.
- **Current usage:** The app header uses `Assets/amplifica-dark-mode.png` (dark blue header background).  
**6. Typography**  
All fonts are available via Google Fonts.  
- **Headings & Titles:** Use  **Barlow Condensed**. Permitted weights are Light, Regular, and Bold.  
- **Body Text & Subtitles:** Use  **Inter**. Permitted weights are Extra Light, Light, Regular, Bold, and Extra Bold.  
- **Taglines:** If pairing a tagline directly next to the logo, it must be Inter Bold and use the Light Gray color (#E1E0E0).  
**7. Logo Constraints**  
- **Clear Space:** The padding/margin around the logo must always be equal to or greater than the height of the lowercase "a" in "amplifica". No other elements can enter this space.  
- **Minimum Sizes (Digital):** Primary Logo (Horizontal) never smaller than 60px in height. Centered Logo (Stacked) never smaller than 130px in height. Secondary Logo never smaller than 70px in height. Isotipo (Icon only) never smaller than 50px in height.  
- **Integrity:** Never distort, modify, or add drop shadows to the logo. The "amplifica" wordmark must *always* read as pure black or pure white.
